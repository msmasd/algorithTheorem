# BOARDCOVER 문제

HxW 크기의 게임판이 있다. 게임판은 검은 칸과 흰 칸으로 구성된 격자모양을 하고있다. 이 중 모든 흰칸을 세칸짜리 L자 모양의 블록으로 덮고싶다.
블록은 회전이 가능하지만, 서로 겹치거나, 검은 칸을 덮거나, 게임판 밖으로 나가면 안된다.
게임판이 주어질 때, 이를 덮는 방법의 수를 계산하시오.

## 조건
2초 안에 실행되어야 하며, 64MB 이하의 메모리를 사용해야 한다.

## 입력
1. 테스트 케이스의 수
2. 두개의 정수(1 <= H, M <= 20)
3. H줄에 각각 W 글자로 게임판이 주어진다.(#은 검은칸, .은 흰칸) (입력에 주어지는 흰칸의 수는 50을 넘지 않는다)

### 예제입력

``` text
3
3 7
#.....#
#.....#
##...##
3 7
#.....#
#.....#
##..###
8 10
##########
#........#
#........#
#........#
#........#
#........#
#........#
##########

출력
0
2
1514
```

## 문제 풀기 전 내가 생각한 방향

재귀처리를 해야할꺼 같다.

모든 흰칸에 대해서 순회하면서 L자 모양이 되는지 확인하고 되면 해당 값 체크 후 재귀
4가지 방법으로 돌리는거 처리.

기저조건은 흰칸의 개수가 3의 배수가 아니면 0 return
모든 흰칸의 값이 없어지면 1 return

## 책 설명

흰 칸의 수를 3으로 나눠서 내려놓을 블록의 수 N을 얻는다.
순서는 중요하지 않지만 완전탐색을 하면 순서에 따라서 개수가 중복되어진다. -> 순회하는 순서를 강제해서 가장 x, y값이 작은 것으로 처리하도록 한다.
